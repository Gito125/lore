## Decisions
- Used `pnpm` as the package manager since `pnpm-lock.yaml` was existing.
- Downgraded `prisma` and `@prisma/client` to version `5.22.0` instead of the newly released `v7.8.0` due to schema syntax breaking changes that aren't fully compatible with Next.js typical `url` definitions in `schema.prisma`.
- Built the UI with `impeccable` design principles: removed AI-like features, added strong typographic scales, custom `clsx` utilities, and Framer Motion spring physics.

## Dependencies Added

## Known Limitations
- The `docker-compose.yml` depends on the user manually running `docker-compose up -d`.
- Prisma migrations will need to be executed on the user's host machine to push the schema changes to their local DB using `npx prisma migrate dev`.
- `auth.ts` uses mock credentials for Google SSO which must be replaced in `.env.local` for production.
