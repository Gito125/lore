## Objective
Finalize the UI/UX polish across the app to achieve the "Dark Editorial" design aesthetic with seamless Framer Motion layout animations and high-impact GSAP moments. Optimize performance using dynamic imports and Next.js best practices.

## Decisions
1. **Route Interception for Layout Animation:** Implementing Framer Motion's `layoutId` for the `ArticleCard -> ArticleDetail` transition in Next.js App Router required using intercepting routes (`@modal/(.)article/[id]`). This allows the feed to remain mounted while the article expands perfectly on top, achieving the Sovereign Archive motion principle.
2. **GSAP Hero Isolation:** The cinematic hero reveal was placed in a dedicated client component (`HeroReveal.tsx`) rendered globally inside the `(main)` layout, checking `sessionStorage` to run only on the user's first visit to `/feed`. This cleanly separates heavy GSAP manipulation from the layout logic.
3. **Dynamic Import for Heavy Data Visualization:** `KnowledgeGraph` was dynamically imported without SSR (`ssr: false`) to avoid shipping heavy D3/GSAP graph libraries in the initial HTML payload, improving TTI.

## Dependencies Added
- None. (Framer Motion and GSAP were already present).

## Known Limitations
- Framer Motion page transitions on standard (non-intercepted) full-page loads still rely on default browser behaviour. The layoutId expand only works effectively via the soft-nav intercepting route, which aligns well with the modal "reading experience".
